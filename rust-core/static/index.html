QES – Quantum Encryption System • architektura prototypu

            
Jak QES funguje

          

          PROTOTYP • SYMETRICKÝ SYSTÉM
        


        
QES je výzkumný prototyp. Není prezentovaný jako hotový auditovaný standard ani jako neprolomitelná vojenská šifra. Je navržený jako symetrický, quantum-hard směřovaný systém: stejná sada tajemství vytváří šifrovací i dešifrovací navigaci.


        

          
Celkový tok

          
TAJNÁ DATA
   │
   ▼
PASSWORD + SEED1–4 + PARTICLE
   │
   ▼
ARGON2ID / KDF
   │
   ▼
MASTER SEED
   │
   ├── hidden_nonce
   ├── hidden_iv
   ├── route_seed
   ├── tag_keys
   └── mac_key
   │
   ▼
QES CORE
permutace → difuze → superpozice → rotace → XOR/ARX prvky → tagové masky
   │
   ▼

   │

   └── Adaptive Labyrinth Cover → cover prostor / další etapa
   │
   ▼
FINÁLNÍ SOUBOR + QES-128 KAPSLE + PUBLIC HASH + KEYED MAC

        


        

          

            
Co je symetrické

            
Symetrické znamená, že stejná tajná sada vstupů slouží pro šifrování i pro návrat zpět. QES nepracuje s veřejným a soukromým klíčem. Pracuje s navigací z passwordu, seedů a particle.

          

          

            
Co není QES Core

            
QES Core není pouze XOR a není pouze ARX. XOR a ARX jsou dílčí vrstvy. Celé jádro je vícevrstvá vratná transformace s permutací, difuzí, superpozicí, rotací, maskami a kontrolou.

          

          

            
Co je kapsle

            
QES‑128 kapsle je samostatný artefakt pro uživatele. Není přilepená k souboru. Neobsahuje heslo ani plaintext. Pomáhá ověřit soubor, režim a správnost klíčové navigace.

          

          

            
Co znamená quantum-hard

            
QES míří na vysokou rezervu proti brute-force útokům: dlouhé tajemství, Argon2id, master seed, MAC a oddělené derivace. Formální tvrzení o bezpečnosti vyžaduje nezávislý kryptografický audit.

   │
   ▼
[1] PERMUTACE
    tajné přeuspořádání dat podle route_seed

   ▼

   │
   ▼

Šifrování a dešifrování jsou symetrické

          
ŠIFROVÁNÍ
password + seedy + particle → master_seed → QES Core → finální soubor + kapsle

DEŠIFROVÁNÍ
stejný password + stejné seedy + stejná particle + kapsle → stejný master_seed → inverse QES Core → původní data

Když se změní jeden vstup:
password / seed / particle / soubor / kapsle / režim
   ↓
MAC nebo TAG nesedí
   ↓
dešifrování se zastaví
